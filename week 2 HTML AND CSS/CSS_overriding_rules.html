Here are the rules that determine which styles override others

  1. Specificity
    • The more specific a selector is, the higher its specificity. Specificity is calculated based on the combination of element type selectors, class selectors, ID selectors, and inline styles.
    • Inline styles (style attribute) have the highest specificity and will override any other styles applied to the same element
    • ID selectors have a higher specificity than class selectors, which have a higher specificity than element type selectors
  2. The !important
    • The !important declaration overrides normal declaration rules. If a style property is marked as !important, it will override any other styles applied to the same element, regardless of specificity.
    • However, it's generally best to avoid using !important as it can make CSS harder to maintain and debug
  3. Order of Appearance
    • If two conflicting style rules have the same specificity and neither rule is marked as !important, the rule that appears later in the CSS file or in the HTML document will override the earlier one.

p { color: blue; /* Rule 1 */ }
.highlight { color: red !important; /* Rule 2 */ }

In Example Mentioned Above

  • The paragraph text is initially styled with blue color by Rule 1.
  • The paragraph has a class of highlight, which applies the red color to the text according to Rule 2. The !important declaration gives this rule higher precedence.
  • Additionally, the paragraph has an inline style that sets the color to green. However, the !important declaration in Rule 2 overrides this inline style.
  • Therefore, the text of the paragraph will be red, as specified by Rule 2.
